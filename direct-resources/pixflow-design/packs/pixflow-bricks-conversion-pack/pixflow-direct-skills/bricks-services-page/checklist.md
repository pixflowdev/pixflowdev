# Bricks Services Page Checklist

## P0

- [ ] Service paths are differentiated by audience, outcome, scope, or next step.
- [ ] The page does not repeat the same card-grid structure across service, benefit, and process sections.
- [ ] Process, proof, objections, or enquiry support is included.
- [ ] No empty media wrappers or placeholder-only cards.
- [ ] Classes are short semantic BEM names.
- [ ] Pixflow variables use valid Bricks object shapes.

## P1

- [ ] One service or offer receives a stronger featured treatment.
- [ ] FAQ or objection copy answers practical concerns.

## P2

- [ ] CTA includes expectation-setting copy.
