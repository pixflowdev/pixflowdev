# Bricks Services Page Reference Checklist

## P0

- [ ] Read the skill, layout references, Bricks element contract, and style-spec contract before planning.
- [ ] Service paths are differentiated by audience, outcome, scope, or next step.
- [ ] Do not list the same services twice with different wording.
- [ ] Include process, proof, objections, or enquiry support.
- [ ] Use short BEM classes and valid Pixflow token object shapes.
- [ ] No empty media wrappers, placeholder-only cards, or unsupported elements.

## P1

- [ ] One service receives a featured editorial treatment.
- [ ] Buying concerns are handled before the final enquiry section.

## P2

- [ ] CTA includes practical expectation-setting copy.
