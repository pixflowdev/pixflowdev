# Bricks Screenshot Redesign Checklist

## P0

- [ ] Screenshot/reference media is used for analysis, not embedded as the final layout shortcut.
- [ ] Main hierarchy, spacing intent, and visual rhythm from the reference are reflected.
- [ ] Final output remains editable Bricks JSON with short BEM classes.
- [ ] No reference-analysis metadata leaks into visible page copy.
- [ ] No inaccessible logos, images, metrics, or proof are invented.
- [ ] Pixflow variables are used for generated styling.

## P1

- [ ] Visual improvements are intentional and not a generic unrelated redesign.
- [ ] Product UI is rebuilt as editable blocks when requested.

## P2

- [ ] Mobile structure is considered, not only the screenshot viewport.
