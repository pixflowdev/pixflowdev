# Bricks Product Spec Skill

Use this hardened pack for product specs, PRDs, requirements, and feature-planning pages. The final output must remain editable Bricks JSON using Pixflow variables, semantic global classes, and short BEM names.

## Workflow

1. Identify the real product spec job: audience, primary task, available facts, missing facts, and desired next step.
2. Select a layout skeleton before writing copy. Do not start from generic hero plus repeated cards.
3. Preserve factual boundaries. Do not invent product decisions, owners, research, timelines, or engineering commitments.
4. Use editable Bricks structures for previews, dashboards, tables, flows, charts, summaries, and placeholders.
5. Run critique before compilation: remove fake proof, repeated card grids, blank media, unsupported live-data claims, and weak hierarchy.

## Output Contract

- Final output is Bricks JSON, not raw HTML, React, native app files, or static screenshots.
- Pixflow Framework tokens and short BEM classes are mandatory.
- Any data values not supplied by the user must be clearly generic placeholders.
