# Bricks Knowledge Workspace Dashboard Checklist

## P0

- [ ] The page matches the requested Pixflow Design template type, not a generic homepage.
- [ ] Every section has a distinct role, hierarchy, and layout family.
- [ ] Claims, metrics, names, dates, prices, policies, and screenshots are not invented.
- [ ] Media and UI mockups are editable Bricks structures unless real media was supplied.
- [ ] Pixflow Framework variables and short BEM classes are used for final Bricks output.

## P1

- [ ] The first viewport has one clear entry point and a practical next step.
- [ ] Support/proof/reassurance appears before the strongest conversion ask.

## P2

- [ ] Mobile layout stays single-column where needed and avoids horizontal overflow.
