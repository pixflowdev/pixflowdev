# Bricks Knowledge Workspace Dashboard Layout References

Use the listed Bricks layout skeletons before inventing a freeform section. Each skeleton must be adapted to the user's prompt and the active Pixflow design system.

## Default Rhythm

- `knowledge_workspace_hero` - use as a distinct section role, not as another equal card grid.
- `workspace_overview` - use as a distinct section role, not as another equal card grid.
- `page_database_panels` - use as a distinct section role, not as another equal card grid.
- `summary_cards` - use as a distinct section role, not as another equal card grid.
- `navigation_cta` - use as a distinct section role, not as another equal card grid.


## Bricks Structure Rules

- Use containers, blocks, headings, text, buttons, icons, images, forms, and code elements only where they are appropriate and editable.
- Use editable UI panels, tables, timelines, previews, and labelled placeholders instead of stock images when no supplied media exists.
- Vary section families so the page does not become repeated centered headings plus cards.
