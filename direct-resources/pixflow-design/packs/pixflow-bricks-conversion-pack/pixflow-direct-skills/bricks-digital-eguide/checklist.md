# Bricks Digital Eguide Checklist

## P0

- [ ] The output matches the requested guide / ebook type, not a generic homepage.
- [ ] The first viewport states the task, audience, and next step without invented proof.
- [ ] Every section has a distinct layout family and role.
- [ ] Tables, panels, previews, charts, and mockups are editable Bricks structures.
- [ ] Do not invent author credentials, download counts, testimonials, or guarantee claims.
- [ ] Placeholder data is labelled or obviously generic when the user did not supply facts.
- [ ] Pixflow Framework variables and short BEM classes are used for final Bricks output.

## P1

- [ ] The page includes a caveat, note, support path, or expectation-setting panel where factual risk exists.
- [ ] Dense information is grouped with clear labels, status chips, or tables rather than equal generic cards.
- [ ] The strongest CTA appears after enough context for the user to decide.

## P2

- [ ] Mobile layout avoids horizontal overflow and keeps tables/panels readable.
- [ ] Repeated lists are visually varied through hierarchy, span, or grouping.
