# Bricks Docs Page Layout Guidance

Use the layouts in layouts.json as the approved skeleton pool. Pick section skeletons before writing copy. Do not use the same centered heading plus equal-card grid repeatedly.

Recommended rhythm:

- `docs_hero`: use `hero_split_visual` with content specific to docs-page.
- `docs_topic_grid`: use `bento_features` with content specific to docs-page.
- `docs_sticky_article`: use `magazine_style_hero` with content specific to docs-page.
- `docs_callout_split`: use `story_split` with content specific to docs-page.
- `docs_support_cta`: use `story_split` with content specific to docs-page.
