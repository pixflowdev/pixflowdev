# Additional Product Brand Checks

## P0

- [ ] Product-brand page plan cannot be accepted with fewer than seven meaningful sections for full homepage/redesign prompts.
- [ ] Screenshot-led FMCG/product-brand prompts cannot select only the generic screenshot-redesign rhythm.
- [ ] If reference media exists, at least two product/brand media roles must survive into section specs.
- [ ] In Pixflow Framework mode, source palette uses brand seed colours only; red/yellow candy brands use yellow first, red second, with white/off-white/dark handled by neutral tint and scheme ramps.

## P1

- [ ] The product hero, category grid, and product showcase each have different visual structure.
- [ ] Quality/trust claims are phrased as supplied or general, not fake-certified.
- [ ] Section colour schemes are assigned semantically: red hero/story, yellow product/category, light editorial/quality, dark mood/CTA where those bands are needed.
- [ ] Heading/body text inherits scheme `--heading`/`--text`; `--foreground` is not used as a text colour.

## P2

- [ ] CTA and footer language matches the product brand, not a software signup.
