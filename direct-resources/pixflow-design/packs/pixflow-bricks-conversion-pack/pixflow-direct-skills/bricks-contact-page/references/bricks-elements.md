# Bricks Contact Page Element Contract

## Allowed Elements

- `section`, `container`, `block`, `div`
- `heading`, `text-basic`, `button`

## Structure

- Use `block` for form-like panels, contact details, support cards, map placeholders, FAQ items, and CTA bands.
- Do not output live form scripts, iframes, or raw embed code from the model.
- Use map/location panels only when the brief provides real location context.

## Bricks Rules

- Use BEM blocks such as `contact`, `details`, `location`, `faq`, and `cta`.
- Keep all text editable in Bricks text/heading elements.
- Use flat responsive overrides for form/detail stacking.
