# Bricks Contact Page Checklist

## P0

- [ ] Contact path is focused and practical.
- [ ] Form or contact panel has expectation-setting copy.
- [ ] No fake addresses, phone numbers, emails, opening hours, or map locations.
- [ ] Form fields and buttons use Pixflow button radius/border tokens.
- [ ] Surface panels use Pixflow surface radius/border tokens.

## P1

- [ ] Reassurance appears near the form.
- [ ] Location/map section only appears when location context exists.

## P2

- [ ] Mobile form layout is simple and does not overflow.
