# Bricks Pricing Page Checklist

## P0

- [ ] No prices, discounts, package names, guarantees, or scarcity are invented.
- [ ] Plans or packages have clear value differences.
- [ ] One recommended/featured option is only used when the brief supports it.
- [ ] Objections or FAQs are close to pricing.
- [ ] The conversion path is clear from every plan/package area.
- [ ] All generated classes are short BEM names.
- [ ] Pixflow variables are used for surfaces, borders, buttons, and spacing.

## P1

- [ ] Pricing, comparison, support, and FAQ use different layouts.
- [ ] Mobile package cards stack without losing CTA visibility.

## P2

- [ ] Final CTA is visually distinct.
