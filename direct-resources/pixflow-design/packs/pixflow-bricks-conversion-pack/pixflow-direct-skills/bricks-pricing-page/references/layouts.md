# Bricks Pricing Page Layout References

## Default Rhythm

1. `pricing_hero` - value frame.
2. `pricing_featured_packages` - packages or plans.
3. `pricing_comparison` - differences and decision support.
4. `pricing_value_support` - reassurance or inclusions.
5. `pricing_objections` - FAQ/objections.
6. `pricing_final_cta` - close.

## Layout Rules

- Do not create fake price numbers.
- Use comparison when plan differences matter.
- Put objections near pricing, not only at the very bottom.
