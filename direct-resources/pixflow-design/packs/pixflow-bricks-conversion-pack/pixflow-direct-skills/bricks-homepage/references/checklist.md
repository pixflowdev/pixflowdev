# Bricks Homepage Reference Checklist

## P0

- [ ] Read the skill, layout references, Bricks element contract, and style-spec contract before planning.
- [ ] Use only Bricks-native elements supported by this compiler pass.
- [ ] Use short BEM global classes and Pixflow token object shapes.
- [ ] Include early proof or trust unless the brief explicitly says not to.
- [ ] Avoid adjacent repeated layout families.
- [ ] Do not invent proof, metrics, testimonials, staff details, or claims.

## P1

- [ ] One middle section has stronger hierarchy than a normal equal-card grid.
- [ ] The final CTA is visually distinct from the previous section.
- [ ] Media or placeholders have a clear role.

## P2

- [ ] Mobile order reads as copy, action, visual, supporting detail.
