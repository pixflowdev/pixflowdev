# Bricks Web Prototype Reference Checklist

## P0

- [ ] Read the skill, layout references, Bricks element contract, and style-spec contract before planning.
- [ ] Choose a section rhythm before writing copy.
- [ ] Output Bricks section/style specs, not raw HTML as the final target.
- [ ] Use Pixflow tokens through Bricks setting shapes.
- [ ] No invented proof, fake metrics, fake logos, or unsupported elements.
- [ ] No adjacent repeated layout families.

## P1

- [ ] Screenshot references influence layout, hierarchy, and spacing without being copied as fragile decorative CSS.
- [ ] Placeholders are honest and labelled.

## P2

- [ ] Mobile order is readable and simple.
