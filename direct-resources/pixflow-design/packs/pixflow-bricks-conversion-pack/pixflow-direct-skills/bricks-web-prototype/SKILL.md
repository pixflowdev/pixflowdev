# Bricks Web Prototype Skill

Use this pack for general website prototypes, screenshot-led redesigns, and one-page mockups that do not fit a more specific page skill.

Follow the Pixflow Design web-prototype discipline: read the design/context, choose a section rhythm before copy, use the layout references, and self-check before output. Unlike Pixflow Design HTML mode, output must become Bricks section specs, Pixflow style specs, and editable Bricks JSON through the compiler.

Do not write raw HTML as the final deliverable. Use Bricks-compatible section structures and scoped style specs.

For Pixflow Stage B workspaces, use the clean-room `bricks-baseline-*` files as the starting Bricks payload when present. The model pass is a targeted audit/refinement pass over that baseline, not a full rebuild from blank.
