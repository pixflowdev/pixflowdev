# Bricks Waitlist Layout References

## Default Rhythm

1. `waitlist_product_hero` - promise and signup.
2. `waitlist_problem_panel` - why now.
3. `waitlist_timeline` - what happens after joining.
4. `waitlist_benefits_bento` - early access value.
5. `waitlist_faq` - objections.
6. `waitlist_signup_cta` - final signup.

## Layout Rules

- Keep the page short and signup-led.
- Use honest placeholders rather than fake launch proof.
- Build product preview as editable Bricks blocks when relevant.
