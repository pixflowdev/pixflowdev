# Bricks Local Service Reference Checklist

## P0

- [ ] Read the skill, layout references, Bricks element contract, and style-spec contract before planning.
- [ ] Show local/service specificity in the opening section.
- [ ] Do not invent addresses, opening hours, reviews, ratings, phone numbers, or credentials.
- [ ] Services and process sections do not use identical card-grid treatments.
- [ ] Contact or booking path has expectation-setting copy.
- [ ] Use valid Bricks token objects and flat responsive override keys.

## P1

- [ ] Proof, reassurance, or practical trust appears before the final CTA.
- [ ] Process copy explains what happens next.

## P2

- [ ] Contact sections stay mobile-first and do not overflow.
