# Bricks Dashboard Interface Checklist

## P0

- [ ] Hero or first major visual contains an editable dashboard/app mockup.
- [ ] UI panels include meaningful labelled states, not blank lines only.
- [ ] Stock photography is not used as a substitute for requested app UI.
- [ ] Sample metrics are labelled as sample or placeholder when not supplied.
- [ ] All generated classes are short BEM names.
- [ ] Pixflow variables are used for generated styling.

## P1

- [ ] Workflow and feature sections use different layout families.
- [ ] Dashboard mockup remains responsive and does not overflow mobile.

## P2

- [ ] Final CTA connects to the product workflow.
