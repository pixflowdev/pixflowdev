# Bricks Dashboard Interface Layout References

## Default Rhythm

1. `dashboard_product_hero` - editable dashboard mockup.
2. `dashboard_workflow_collage` - workflow states.
3. `dashboard_feature_matrix` - capabilities.
4. `dashboard_process_flow` - how work moves through the product.
5. `dashboard_objections` - practical questions.
6. `dashboard_final_cta` - conversion.

## Layout Rules

- Use dashboards, nav rails, chart placeholders, settings panels, and table rows as editable blocks.
- Do not use stock photos for product UI.
- Avoid meaningless generic data.
