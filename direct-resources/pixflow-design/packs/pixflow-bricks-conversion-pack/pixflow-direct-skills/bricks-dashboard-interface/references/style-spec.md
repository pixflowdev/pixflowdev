# Bricks Dashboard Interface Style Spec Contract

## Token Rules

- Mockup surfaces use `var(--foreground)`.
- Borders use `var(--border)` and `max(var(--border--btn), 1px)`.
- UI frames use `var(--radius--surface)`.
- Buttons and chips use `var(--radius--btn)`.

## Layout Style Rules

- Keep UI panels inspectable and dense enough to read as software.
- Use accent for active state, chart mark, or primary CTA.
- Avoid overdecorated app mockups that become hard to edit in Bricks.
