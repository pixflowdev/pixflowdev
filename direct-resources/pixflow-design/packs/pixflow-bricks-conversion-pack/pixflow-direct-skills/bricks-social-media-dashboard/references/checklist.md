# Bricks Social Media Dashboard Reference Checklist

## P0

- [ ] Use the supplied facts first; do not fabricate names, dates, metrics, claims, owners, or testimonials.
- [ ] Include at least one structured, non-card-grid information surface: table, matrix, timeline, dashboard panel, or document block.
- [ ] Add an explicit caveat/assumption/support block when data is missing.
- [ ] Keep all visual previews editable in Bricks unless the user supplied actual media.
- [ ] Preserve the selected skill's default rhythm before adding generic sections.
- [ ] Use Pixflow token classes and no raw hardcoded colour palette.
