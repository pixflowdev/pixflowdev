# Paper Accessibility

- Maintain WCAG 2.2 AA contrast. Muted copy on off-white paper must remain readable.
- Do not use blue text on pale blue backgrounds unless contrast is checked.
- Focus-visible states must be clear on links, buttons, cards used as links, inputs, and tabs.
- Touch targets should be at least 44px high where interactive.
- Editor mockup controls can be illustrative, but any real buttons or links need labels.
- Do not rely on texture, color, or tiny dots alone to communicate state.
- Use semantic landmarks: `header`, `nav`, `main`, `section`, `article`, `aside`, `footer`.
- If using motion, support `prefers-reduced-motion`.
- Pricing copy should not imply real billing unless the user provides actual pricing.
- Mobile reading order should match visual order.
