# Luxury Responsive Rules

- Desktop can use large vertical space, but mobile must not feel like a cropped desktop.
- Desktop review target is 1600px wide with a primary content max around 1440px.
- At desktop, test every two-column section for compression. Repair when a heading becomes one-word-per-line, a paragraph reads like a narrow sidebar, or an image/card cannot breathe.
- Valid repairs include full-width intro plus grid, centered intro, asymmetric columns, moved body copy, smaller local heading scale, or a one-column stack.
- Do not automatically stack every two-column section; keep the split when both tracks are useful and readable.
- Keep hero type readable at 360px viewport width.
- Use explicit max widths for uppercase headings.
- Collapse image stacks to one dominant frame plus one optional small tile.
- Stack editorial splits with image before text when image is the primary context.
- Reduce section padding on mobile while preserving rhythm.
- Keep buttons full-width only when it improves tap comfort.
- Check that pill labels, nav links, and product metadata never overflow.
