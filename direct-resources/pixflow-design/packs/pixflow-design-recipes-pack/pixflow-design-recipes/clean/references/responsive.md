# Clean Responsive Rules

## Desktop

- Review at 1600px wide.
- Use a primary content max width around 1180px to 1280px.
- Use up to 1440px only for large product slabs, integrations, or wide proof sections.
- Center the hero and keep the H1 to two to four readable lines.
- Do not let feature cards become narrow sidebars by accident.

## Tablet

- Collapse complex asymmetric grids before cards become cramped.
- Product slabs can remain wide but should reduce internal density.
- Header center navigation can be hidden while account actions remain visible.
- Pricing cards may move from three columns to two, then one.

## Mobile

- Stack CTAs at full width when two buttons would become cramped.
- Product slabs should remain inspectable. Remove nonessential sidebars/tabs before shrinking text too far.
- Feature cards, pricing cards, testimonial cards, and large proof cards stack to one column.
- Integration tiles can use two columns if labels remain readable.
- Keep mobile section padding generous but not theatrical.

## Overflow Checks

- No horizontal overflow from product mockups, app buttons, badges, code rows, or long labels.
- Long headings should wrap naturally and not create one-word vertical stacks unless the brief explicitly asks for that.
- Pricing badges and status chips must stay inside cards.
- Code/mockup text can truncate only when it still looks intentional and readable.

## Repair Rules

- If a two-column section compresses content, widen the text track, stack the intro, or move media below.
- If a grid creates thin cards, reduce the number of columns rather than reducing font size.
- If the hero feels empty, add a trust row or bring product proof closer.
- If mobile becomes too long, combine repeated feature cards or use compact rows.
