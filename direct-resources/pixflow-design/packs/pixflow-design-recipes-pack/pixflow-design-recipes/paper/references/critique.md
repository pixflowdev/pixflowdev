# Paper Critique

Score each dimension 1-5 before final handoff. Repair anything below 4.

## Philosophy

Does the artifact feel like a modern light document workspace with tactile paper surfaces?

Failure examples:

- It looks like a generic SaaS page with a dashboard.
- It looks like an editorial beige report.
- It follows dark mode without an explicit user request.

## Hierarchy

Is the hero calm, readable, and supported by a strong editor/workspace proof object?

Repair:

- Increase editor shell scale.
- Stack the hero to avoid squeezed columns.
- Tighten headline line breaks.

## Execution

Are surfaces, borders, radii, miniatures, CTAs, and section spacing consistent?

Repair:

- Normalize radii and borders.
- Replace decorative icons with product miniatures.
- Remove inconsistent shadows or gradients.

## Specificity

Does the page contain concrete document/editor/collaboration details?

Repair:

- Add document titles, sidebar groups, comments, permission rows, version history, search, code/docs previews, or publish workflows.
- Rewrite vague benefit copy into product behavior.

## Restraint

Is the palette light, quiet, and controlled?

Repair:

- Reduce accent usage.
- Remove decorative textures.
- Keep dark only for a highlighted plan/footer if useful.

## Responsive

Does it work at desktop, tablet, and mobile?

Repair:

- Collapse squeezed splits.
- Stack cards and workflow steps.
- Simplify editor shell.
- Ensure all controls fit.
