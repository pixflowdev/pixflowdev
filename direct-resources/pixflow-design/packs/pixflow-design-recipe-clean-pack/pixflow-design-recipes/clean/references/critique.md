# Clean Critique

Use this after the first artifact draft and before handoff.

## First Viewport

- Can the product outcome be understood in five seconds?
- Is the hero centered, spacious, and specific?
- Are the CTAs clear and short?
- Is there enough trust/product context near the first viewport?

## Product Evidence

- Is the main product proof inspectable, or does it look like decoration?
- Are labels, tabs, rows, or states relevant to the brief?
- Does the product proof have a stable aspect ratio and enough width?
- Does it avoid unreadable tiny UI?

## Section Rhythm

- Does each section have one job?
- Does the page alternate white and soft surfaces with restraint?
- Is there a commercial support section when the route needs it: pricing, testimonials, integrations, security, or compatibility?
- Is there only one strong contrast/accent moment?

## Width And Layout

- At 1600px, are cards and text columns wide enough?
- Has any grid created extra columns that make cards too thin?
- Does the content use the desktop width deliberately?
- Does mobile stack cleanly without overflow?

## Style Integrity

- Does it still feel Clean below the hero?
- Is typography sans-serif, direct, and readable?
- Is accent usage purposeful and limited?
- Are shadows subtle?
- Is there any external source branding in visible copy? If yes, remove it.

## Repair Instructions

- If the page feels empty, add product proof, logos, integrations, or testimonials.
- If the page feels generic, add route-specific UI labels, states, and workflows.
- If the page feels noisy, remove decorative cards and reduce accent surfaces.
- If columns are cramped, stack or widen rather than shrinking text.
- If the preview looks like a card demo, expand it into a complete landing page.
