# Bricks SaaS Landing Checklist

## P0

- [ ] Header, hero, proof, product highlight, template/library or product preview, feature matrix, and final CTA are present when requested.
- [ ] Hero includes an editable browser/product mockup, not generic office photography.
- [ ] Product UI elements are built as editable Bricks blocks unless a supplied screenshot is explicitly used.
- [ ] No invented metrics, ratings, customer counts, logos, or testimonials.
- [ ] Sections do not collapse into repeated centered heading plus equal card grid.
- [ ] All section/card/image/button styling uses Pixflow token object shapes.
- [ ] Classes remain short semantic BEM names.

## P1

- [ ] First viewport has one clear headline, compact supporting copy, and the product visual above the fold.
- [ ] Product proof appears immediately after the hero.
- [ ] Feature sections vary between collage, template preview, bento/matrix, and CTA.

## P2

- [ ] Product UI mockups include realistic chrome, panels, structure, and canvas details.
- [ ] Template preview cards have visual thumbnails, not text-heavy bodies.
