# Bricks Dynamic Data

- Bricks dynamic data uses token syntax such as post/user/field tags in text-capable controls.
- Provider-specific tags can come from ACF, Meta Box, JetEngine, Pods, WooCommerce, and other plugins.
- Do not invent dynamic-data provider tags. Verify them from the live site before using them.
- If a page needs CMS content but tags are unknown, compile honest editable placeholders and mark the intended dynamic role in diagnostics.
- Dynamic data should not be used for static marketing copy unless the user specifically asks for CMS-driven content.
